# 客戶案件管理系統 (Customer Case Management System)

本專案旨在為保險與金融業務同仁打造一個高效、直覺且符合銷售心理學的客戶與案件管理系統。

## 👥 角色分工框架
在開發過程中，我將融合以下三種角色，輔導您完成開發：
1. **業務同仁 (Sales/Business Expert)**：確保系統功能符合台灣保險與金融市場實務需求，融入銷售心理學（如 UMBC / CARE 心理架構），提升客戶拜訪與提案成交率。
2. **系統工程師 (System Engineer)**：負責系統架構設計、資料庫規劃、前後端邏輯與代碼實作，確保系統乾淨、模組化且易於維護。
3. **系統整合師 (System Integrator)**：負責各功能模組的串接（如 Google Sheets API、本機快取與 RWD 介面整合），確保系統在平板與手機上都能流暢展示。

## 🪙 Token 節省開發策略 (Token-Saving Strategy)
為了避免燒掉過多的 Token，我們將嚴格執行以下原則：
* **模組化拆解**：不寫龐大臃腫的單一檔案。將 HTML, CSS, JS 分離，或將功能拆解成獨立的小組件，每次只修改受影響的模組。
* **分步確認**：每一步只聚焦於「一個核心問題」（如：資料庫欄位設計、首頁版面配置、單一功能的邏輯），在您確認後才進行代碼撰寫，避免重寫造成的 Token 浪費。
* **文件化狀態**：利用本 README 與任務看板（task.md）紀錄目前的狀態與設計決定，避免在對話歷史中反覆回顧。

## 🧠 ADHD 友善設計原則 (ADHD-Friendly UX)
針對成人 ADHD 的使用者特性，本系統在介面與操作上將貫徹以下四大設計哲學：
1. **抗「時間盲感」 (Time Blindness)**：使用強烈且直覺的「色彩編碼與呼吸燈效果」，第一時間突顯「今天必做」與「已逾期」案件，免去大腦重新運算時間的負擔。
2. **降低「認知負載」 (Reduce Cognitive Load)**：拒絕密密麻麻的輸入表格。操作流程一律採用「一鍵完成/自動帶入」，例如點擊亮燈即自動記錄「今天」為成交日，減少手動選擇日曆的繁瑣步驟。
3. **減少「工作記憶」負擔 (Support Working Memory)**：拒絕深層彈窗。將核心操作（如備註紀錄、快速改期）整合在畫布的最上層，讓大腦隨時看得到當前上下文。
4. **微多巴胺回饋 (Micro-Dopamine Hit)**：案件推前（例如拖曳卡片或成交亮燈）時，加入精緻的微型動畫與視覺效果，提供即時的心理回饋與成就感，維持使用動能。
