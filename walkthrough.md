# 客戶 360° 顧問畫布 & 三大系統無縫跳轉 Walkthrough

我們已成功實作並交付了全新的客戶關係顧問系統！這將您的應用程式演進為一個能夠同時應對「專案追蹤 (案件)」與「家庭深耕 (客戶)」的完整生態圈。

## 變更內容說明

### 1. 「固定訊息清單」與「保險產品清單」雲端資料庫同步整合與拖曳排序
* **升級項目**：
  * **後端架跡升級**：在 `code.gs` 裡動態新增 `FixedMessages` 工作表，並補齊 `Products` 表覆蓋時漏掉的 `period` (年期) 欄位處理。同時擴建 API (`doGet` 與 `doPost`) 處理這兩個表。
  * **前端同步引擎**：在 `app.js` 的 `init` 與存檔流程中，加入 `fetchProductsFromCloud()`、`saveProductsToCloud()`、`fetchFixedMessagesFromCloud()` 以及 `saveFixedMessagesToCloud()`。現在，這兩個輔助工具的資料庫會跨電腦自動即時同步。
  * **固定訊息拖曳排序**：固定訊息左側清單支援標準 HTML5 拖曳排序（Drag & Drop），並藉由 polyfill 支援平板/觸控裝置。拖曳時游標會自動呈現手掌抓取（grab/grabbing）狀態，排序後會即時同步存入 LocalStorage 與 Google Sheets 雲端。
  * **細節與視覺優化**：移除了非必要的「步驟 6 - 」序號字眼、卡片裝飾 emoji 與停用標記，保額相容任意字串。統一使用全系統一致的 `✏️` 與 `🗑` 按鈕。移除了多餘 of 重複新增按鈕。
  * **清單字體顏色亮白化**：針對產品複習列表中的參數字體（如性別、年齡等）與固定訊息清單標題字體，將原先較暗的灰色一律改為**高對比、明亮好辨識的純白色**。
  * **移除了停用項目的半透明淡化 (解決灰色問題)**：為了解決已停用 (inactive) 產品在左側清單因 `opacity: 0.5` 被瀏覽器淡化成灰色的問題，我們移除了該項目的透明度減半機制。現在，不論啟用或停用，清單內的所有文字與參數皆能以 **100% 亮度的純白色**呈現，提供直覺且最舒適的閱讀焦點。
  * **待辦事項指定姓名黏著化**：當待辦任務有指定客戶姓名時，系統會直接在任務標題（Title）後方以橘黃色高亮字體黏著呈現 `(客戶姓名)`，不論是**待辦管理頁面**還是首頁的**提醒通知彈窗面板**都會顯示，幫助 ADHD 業務員 0.1 秒精準辨別任務對象！
  * **點擊「產壽險徽章」移至手動排序最前端**：在主幹道最左側的「產 / 壽」徽章中新增了點擊事件。當業務員經由搜尋欄篩選或在任何排序下，只要點一下該案件最左側的「壽」或「產」徽章，該案件便會**立即移至手動排序的第一位（最前端）**，同時系統會自動將排序模式切換回「🖐️ 手動排序」並即時存檔。這讓業務員在清除搜尋後，能第一時間看見該被置頂處理的案件。
  * **增員輔導管理主面板重構為雙欄視角**：
    * **左側版面**：新增了三大方塊按鈕（目前皆先寫「準備工作」），高度大小整齊網格化對齊，方便點選切換；按鈕下方為垂直排列的增員對象清單，擺脫原本分散的卡片網格佈局。
    * **右側版面**：當點選三大準備工作方塊時，右側會呈現對應的文字編輯/維護介面，可即時儲存內容至本機中（方便未來放入五大工作動力或工作思維等規範）；當點選左側下方增員名單時，右側會改為顯示該增員對象詳情，並提供「🧭 增員路徑」（含基本資料與面談進度）與「📝 輔導紀錄」兩大子頁籤切換，將資訊扁平置於同一頁面，解決 ADHD 業務員頻繁開關 Drawer 抽屜的認知負擔。
  * **新增「銷售流程定義」維護分頁**：
    * **後端架構**：在 `code.gs` 裡動態新增 `SalesProcesses` 工作表與 headers (`id`, `stage`, `task`, `intent`, `document`)，支援整批覆蓋式同步寫入與讀取。
    * **獨立拖曳排序**：流程清單支援標準 HTML5 拖曳排序，並整合觸控裝置 polyfill。順序隨意調整，拖曳後即時同步存入 LocalStorage 與 Google Sheets 雲端。
    * **高清晰展示與編輯**：點選左側流程項目即可在右側快速檢視「階段」、「任務」、「用意」與「文件」；點擊「✏️ 編輯流程」可進入無干擾編輯模式，不與本系統做任何連動，適合作為獨立的銷售指導原則與備忘定義。
  * **跨裝置手動與背景熱同步**：
    * **強制拉取按鈕**：維護中心頂部 Header 右側新增「🔄 雲端同步」按鈕，點選即時從雲端試算表拉取最新產品、固定訊息與流程定義，並自動更新本地快取重新渲染。
    * **背景自動同步**：加入 3 分鐘定時輪詢，自動在背景向 Google Sheets API 發送異步 fetch 請求，若使用者已打開維護中心介面，將在不干擾操作的前提下偷偷在背景完成清單更新，完美達成雙機熱備同步！
  * **輸入框聚焦 Esc 防誤關機制**：
    * 全域鍵盤監聽（Keyboard Listener）新增了防護。當使用者的游標焦點停留在 `input`、`textarea` 或是任何具有 `contenteditable` 屬性的文字輸入區時，按下鍵盤 `Esc` 鍵只會收起或取消中文輸入法的「選字詞小視窗」，而**絕對不會**觸發關閉開工六步驟（提醒彈窗）或是其他背景 Modal，從根本上解決了中文輸入與快捷鍵的衝突，防止編輯中斷。
  * **修復銷售流程同步 Payload 傳參格式**：
    * 針對同步銷售流程到 Google Sheets 雲端時報出的 `SyntaxError: Unexpected token 'a' in JSON` 錯誤進行修復。將 `saveSalesProcessesToCloud` 發送的數據封裝與 Content-Type 改為全系統一致的標準 JSON 模式，修正了與 Google Apps Script 後端 `JSON.parse` 之間的解析衝突。現在，**每次新增、修改完成並點選「儲存定義」時，皆能 100% 完全自動即時寫入雲端**，不需要手動按下同步按鈕。
  * **維護中心極簡與按鈕動態收納**：
    * **刪除手動雲端同步**：移除了 Header 中的手動同步按鈕，因為點擊儲存時已實現 100% 背景自動熱備同步。
    * **按鈕動態收納**：在銷售流程定義編輯/修改時，頂部的 `＋ 新增流程` 按鈕會動態替換為 `儲存定義` 和 `取消`，並刪除了底部原本多佔一整行的按鈕列，介面整潔且緊湊。

### 2. 徹底修復跨階段約定導致今日到訪漏算問題 (超強聯集判定)
* **漏洞原因**：
  * 當前案件 `許文曦` 大階段處於 `SA` (主畫面上 SA 燈亮)。
  * 但業務員已經成功跟客戶約好了下一個階段初訪的日期為今天 `2026-07-15`，因此日期被填在 `oaDetails.meetDate` 中。
  * 舊有的判定器會因為 `c.currentPhase === 'SA'`，只去核對 `saDetails.agreeDate`。既然 `saDate` 填寫的是過去或為空，便會判定該案件今天沒約，導致被漏計。
* **解決方案**：
  * 我們引進了**「五階段約訪日期全聯集判定」**。
  * 現在，系統會直接提取案件在所有流程節點（SA / OA / PC / C / S）填寫的約定日期，只要**其中任何一個日期等於今天 (07/15)，該案件就 100% 被判定為今日到訪**！
  * 這完全擺脫了案件當前處於什麼階段的限制，徹底根治了跨階段約定、跨欄位排程漏算的所有可能性。
